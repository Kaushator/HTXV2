## V. Советы Copilot для ускорения и экономии токенов

- Использовать SRP: один компонент = одна задача.
- Все типы, мок-данные, утилиты — в отдельные файлы.
- Краткие комментарии-пояснения Copilot перед компонентом (см. предыдущий пример).
- Не плодить дубликаты, использовать хуки и shared utils.
- Для сложных UI — сначала скелет/структура, детали — отдельными задачами.
- Минимизировать дублирование props/типов, использовать auto-import.
