# Model Context Protocol (MCP) для HTXV2

Этот каталог содержит конфигурацию и инструменты для работы с Model Context Protocol в проекте HTXV2.
MCP расширяет возможности GitHub Copilot Chat и позволяет взаимодействовать с системой HTX непосредственно из редактора кода.

## Настройка MCP сервера

### Автоматическая настройка (рекомендуется)

1. Откройте проект в VS Code с использованием dev-контейнера:

   - Нажмите F1 и введите "Remote-Containers: Reopen in Container"
   - Инициализация MCP произойдет автоматически

2. Убедитесь, что расширения GitHub Copilot и GitHub Copilot Chat установлены

   - При необходимости установите их из маркетплейса VS Code

3. Перезагрузите окно VS Code (F1 → `Developer: Reload Window`)

### Ручная настройка

1. Установите NodeJS 18+ и npm
2. Выполните в корне проекта:

   ```bash
   npm install
   npm run dev
   ```

3. Проверьте, что в `.vscode/settings.json` указаны правильные настройки MCP

## Проверка работоспособности

1. Откройте Copilot Chat и наберите:

   ```text
   /tools
   ```

2. Должен появиться список инструментов MCP:

   - `health`: Проверка состояния системы
   - `list_assets`: Просмотр доступных активов
   - `analysis`: Запуск аналитики данных
   - `upload_csv`: Загрузка CSV файлов для анализа
   - И другие специфичные для проекта инструменты

3. Если инструменты не отображаются:
   - Проверьте, запущен ли MCP-сервер
   - Обновите расширения Copilot до последней версии
   - Проверьте пути и настройки в `.vscode/settings.json`

## Возможности MCP в HTXV2

MCP добавляет следующие функции в GitHub Copilot Chat:

1. **Анализ данных**:

   - Обработка финансовых данных
   - Построение аналитических отчетов
   - Визуализация результатов

2. **Управление контейнерами**:

   - Проверка состояния контейнеров
   - Перезапуск и обновление сервисов
   - Диагностика проблем развертывания

3. **Работа с базой данных**:
   - Выполнение запросов
   - Управление схемой
   - Резервное копирование и восстановление

## Настройка и конфигурация

- Переменные окружения определены в блоке `env` в `.vscode/settings.json`
- Рабочая директория задается через `cwd` (по умолчанию: `${workspaceFolder}`)
- При изменении структуры проекта обновите параметры `cwd` и `args`

## Разработка MCP-инструментов

Для добавления новых инструментов в MCP:

1. Создайте новый файл в каталоге `src/tools/`
2. Зарегистрируйте инструмент в `src/tools/index.ts`
3. Перезапустите MCP-сервер: `npm run dev`

Подробная документация по разработке инструментов MCP находится в [официальной документации GitHub](https://github.com/github/dev/blob/main/doc/mcp.md)
