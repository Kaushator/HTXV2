# Инструкция по настройке GitHub Copilot с MCP для HTXV2

## Обзор

HTXV2 поддерживает интеграцию с GitHub Copilot через MCP (Model Context Protocol), что позволяет:

1. Получать интеллектуальные подсказки по коду с учетом специфики проекта
2. Запрашивать данные рынка и портфеля через интерфейс чата
3. Анализировать торговые сигналы и PnL метрики
4. Генерировать код с учетом архитектуры проекта

## Шаги настройки

### 1. Установка зависимостей

```bash
# Установка зависимостей MCP сервера
npm install
```

### 2. Сборка проекта

```bash
# Сборка TypeScript кода
npm run build
```

### 3. Запуск MCP сервера

Запустите MCP сервер одним из способов:

```bash
# Через Node.js
node start.js

# Или через скрипт PowerShell
./start-mcp.ps1
```

### 4. Проверка работоспособности

Убедитесь, что MCP сервер работает корректно:

```bash
# Проверка статуса сервера
curl http://localhost:3001/status
```

Результат должен быть подобным:

```json
{
  "status": "ok",
  "uptime": 12.345,
  "timestamp": 1631234567890,
  "redisConnected": true
}
```

### 5. Использование в VS Code

После запуска MCP сервера, GitHub Copilot будет автоматически использовать его для улучшения подсказок.

Вы можете проверить статус подключения в VS Code:

1. Откройте палитру команд (`Ctrl+Shift+P`)
2. Введите "GitHub Copilot: Diagnostics"
3. Проверьте статус MCP подключения

## Возможности MCP интеграции

### Рыночные данные

- Получение текущих котировок
- Анализ исторических данных
- Визуализация графиков

### Портфель

- Анализ состояния активов
- Расчет PnL метрик
- Отслеживание операций

### Торговые сигналы

- AI рекомендации
- Технические индикаторы
- Статистика эффективности

## Решение проблем

Если MCP интеграция не работает:

1. Проверьте, запущен ли MCP сервер
2. Убедитесь, что настройки VS Code содержат корректную конфигурацию MCP
3. Проверьте логи VS Code (View > Output > GitHub Copilot)
4. Перезапустите VS Code и MCP сервер
