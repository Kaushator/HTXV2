# Настройка GitHub Actions для HTXEnterface_v2

Этот документ описывает настройку GitHub Actions для проекта HTXEnterface_v2, включая настройку Copilot и другие рабочие процессы CI/CD.

## Шаги настройки Copilot

Файл `.github/workflows/copilot-setup-steps.yml` содержит настройки для интеграции GitHub Copilot с нашим проектом. Этот workflow автоматически настраивает окружение для работы Copilot с нашим кодом.

### Возможности workflow Copilot

- Автоматически запускается при изменении файла настройки
- Может быть запущен вручную через интерфейс GitHub Actions
- Настраивает необходимую среду выполнения для работы с кодом:
  - Node.js 20 для фронтенда
  - Python 3.11 для бэкенда
  - Redis и PostgreSQL для тестирования
  - Docker для сборки контейнеров
- Выполняет миграции базы данных

### Как использовать

1. Workflow автоматически запускается при изменении файла `.github/workflows/copilot-setup-steps.yml`
2. Можно запустить вручную через интерфейс GitHub Actions
3. После успешного выполнения Copilot будет настроен для работы с проектом

## Другие рабочие процессы CI/CD

Проект также содержит следующие рабочие процессы GitHub Actions:

- `ci-cd.yml`: Основной CI/CD pipeline для тестирования, сборки и деплоя
- `status-report.yml`: Генерирует отчеты о состоянии проекта

## Скрипты для работы с GitHub

В проекте доступны следующие скрипты для работы с GitHub:

1. **Push-ToGitHub.ps1**: PowerShell скрипт для отправки изменений в GitHub

   ```powershell
   .\Push-ToGitHub.ps1 -Message "Добавлен скрипт настройки Copilot"
   ```

2. **push-to-github.sh**: Bash скрипт для отправки изменений в GitHub

   ```bash
   ./push-to-github.sh -m "Добавлен скрипт настройки Copilot"
   ```

### Параметры скриптов

- `-m`, `--message`: Сообщение коммита
- `-b`, `--branch`: Ветка для отправки изменений
- `--skip-format`: Пропустить форматирование кода
- `-f`, `--force`: Использовать флаг force при push

## Рекомендации по работе с GitHub

1. **Ветки**: Используйте отдельные ветки для разных функций и исправлений
2. **Коммиты**: Пишите информативные сообщения коммитов
3. **Pull Requests**: Создавайте PR для слияния изменений в основную ветку
4. **Code Review**: Используйте функцию Code Review для проверки изменений
5. **GitHub Actions**: Следите за результатами выполнения GitHub Actions
