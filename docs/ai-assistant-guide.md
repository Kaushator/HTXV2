# Инструкции для ИИ-ассистентов по работе с проектом HTXV2

## Общая информация о проекте

HTXV2 (HTX Interface v2) - это интерфейс взаимодействия с системой HTX, обеспечивающий доступ к функциям анализа и обработки данных. Проект имеет микросервисную архитектуру и состоит из следующих основных компонентов:

1. **Frontend**: React/Next.js приложение, предоставляющее пользовательский интерфейс
2. **Backend**: FastAPI (Python) сервис, предоставляющий REST API
3. **FinGPT**: Специализированный сервис для финансового анализа на базе языковых моделей
4. **PostgreSQL**: База данных для хранения информации о пользователях, сессиях и настройках
5. **Redis**: Кэширование и временное хранение данных сессий

## Рабочий процесс для ассистентов

### 1. Анализ кода и структуры

При работе с проектом сначала изучите структуру директорий и файлов для понимания архитектуры. Обратите внимание на:

- Структуру маршрутов API в backend/app/api/
- Модели данных в backend/app/models/
- Компоненты frontend в frontend/src/components/
- Страницы frontend в frontend/src/pages/ и frontend/app/

### 2. Запуск и тестирование

Для локального запуска приложения используйте Docker Compose:

```bash
docker compose up -d
```

или запустите задачи VS Code:

- "Docker: compose up" для запуска всех контейнеров
- "Backend: Run Dev Server" для запуска бэкенда
- "Frontend: Run Dev Server" для запуска фронтенда

### 3. Основные эндпоинты API

- **Аутентификация**:

  - POST `/api/auth/login` - вход пользователя
  - POST `/api/auth/register` - регистрация пользователя
  - POST `/api/auth/logout` - выход
  - GET `/api/auth/me` - информация о текущем пользователе

- **Основные операции**:
  - GET `/api/data/analysis` - получение результатов анализа
  - POST `/api/data/upload` - загрузка данных для анализа
  - GET `/api/settings` - получение настроек
  - PUT `/api/settings` - обновление настроек

### 4. Работа с базами данных

- **PostgreSQL**: Основная база данных для хранения данных

  - Используйте SQLAlchemy для ORM-операций
  - Миграции управляются через Alembic

- **Redis**: Используется для кэширования и временного хранения
  - Сессии пользователей
  - Результаты запросов
  - Промежуточные данные анализа

### 5. Типичные задачи разработки

#### Frontend

- Добавление новых компонентов в `/frontend/src/components/`
- Создание новых страниц в `/frontend/app/`
- Интеграция с API через сервисы в `/frontend/src/services/`
- Обновление стилей с использованием TailwindCSS

#### Backend

- Добавление новых эндпоинтов в `/backend/app/api/`
- Создание моделей данных в `/backend/app/models/`
- Обновление схем Pydantic в `/backend/app/schemas/`
- Добавление бизнес-логики в `/backend/app/services/`

### 6. Утилитные скрипты

В проекте доступны следующие утилитные скрипты для разработки:

- **test-containers.ps1** / **test-htx** - тестирование контейнеров
- **update-containers.ps1** / **update-htx** - обновление контейнеров
- **backup-data.ps1** / **backup-htx** - создание резервных копий данных

### 7. Типичные ошибки и их решения

#### Ошибка подключения к базе данных PostgreSQL

**Симптом**: Сервер не запускается с ошибками подключения к PostgreSQL

**Решение**:

1. Проверьте наличие баз данных: `docker exec -it htxenterface_v2-postgres-1 psql -U htx -d htxdb -c "\l"`
2. Создайте недостающие базы: `docker exec -it htxenterface_v2-postgres-1 psql -U htx -d htxdb -c "CREATE DATABASE htx;"`
3. Проверьте строки подключения в `backend/app/core/config.py`

#### Ошибки сборки frontend

**Симптом**: Ошибки сборки Next.js приложения

**Решение**:

1. Проверьте установленные зависимости: `cd frontend && npm list`
2. Удалите node_modules и переустановите: `cd frontend && rm -rf node_modules && npm ci`
3. Проверьте совместимость версий пакетов в package.json

### 8. Рекомендации по оптимизации и улучшению кода

1. **Performance**:

   - Используйте React.memo() для предотвращения ненужных перерендеров
   - Оптимизируйте SQL запросы (используйте индексы, limit, offset)
   - Применяйте кэширование для часто запрашиваемых данных

2. **Безопасность**:

   - Валидируйте все входные данные через Pydantic модели
   - Используйте параметризованные запросы для предотвращения SQL-инъекций
   - Применяйте HTTPS для всех внешних соединений

3. **Масштабируемость**:
   - Используйте асинхронные обработчики в FastAPI
   - Реализуйте горизонтальное масштабирование сервисов
   - Оптимизируйте работу с базой данных через connection pooling

## Особенности окружения разработки

### Dev-контейнер

Проект настроен для работы с VS Code dev-контейнерами. При открытии проекта в VS Code с установленным расширением Remote - Containers, среда разработки будет автоматически настроена.

Для инициализации dev-контейнера выполните:

- `.devcontainer/scripts/init-dev-container.sh` (для Linux/macOS)
- `.devcontainer/scripts/powershell/init-dev-container.ps1` (для Windows/PowerShell)

### Полезные VS Code расширения

Для работы с проектом рекомендуются следующие расширения VS Code:

- ms-python.python
- ms-python.vscode-pylance
- esbenp.prettier-vscode
- dbaeumer.vscode-eslint
- ms-vscode.powershell
- ms-azuretools.vscode-docker
