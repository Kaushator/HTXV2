# Руководство по инициализации dev-контейнера HTXV2

## Введение

Этот документ описывает процесс инициализации и настройки development-контейнера для проекта HTXV2. Dev-контейнер предоставляет изолированную и предварительно настроенную среду разработки, которая включает все необходимые инструменты и зависимости для работы с проектом.

## Структура dev-контейнера

Dev-контейнер включает следующие компоненты:

- Базовый образ на основе Ubuntu с предустановленными инструментами разработки
- PowerShell Core для кросс-платформенных скриптов
- Docker и Docker Compose для управления контейнерами приложений
- Набор утилитных скриптов для тестирования, обновления и резервного копирования
- Автоматическую настройку баз данных PostgreSQL

## Скрипты инициализации

В проекте доступны следующие скрипты инициализации:

1. **Bash (Linux/macOS)**: `.devcontainer/scripts/init-dev-container.sh`
2. **PowerShell (Windows/Linux/macOS)**: `.devcontainer/scripts/powershell/init-dev-container.ps1`

### Что делают скрипты инициализации

Скрипты инициализации выполняют следующие действия:

1. Проверяют наличие и работоспособность Docker и Docker Compose
2. Запускают контейнеры, если они не запущены
3. Проверяют и создают необходимую базу данных `htx` в PostgreSQL
4. Настраивают alias-команды и символические ссылки для удобства работы
5. Тестируют доступность frontend и backend сервисов
6. В Bash-версии устанавливаются дополнительные утилиты и настраивается окружение

## Использование

### Bash (Linux/macOS)

```bash
# Перейдите в корневую директорию проекта
cd /path/to/HTXV2

# Запустите скрипт инициализации
.devcontainer/scripts/init-dev-container.sh
```

### PowerShell (Windows/Linux/macOS)

```powershell
# Перейдите в корневую директорию проекта
cd path\to\HTXV2

# Запустите скрипт инициализации
.\.devcontainer\scripts\powershell\init-dev-container.ps1
```

## Утилитные команды

После инициализации dev-контейнера доступны следующие утилитные команды:

| Команда      | Описание                                                 |
| ------------ | -------------------------------------------------------- |
| `test-htx`   | Проверка состояния контейнеров и их доступности          |
| `update-htx` | Обновление контейнеров (pull новых образов и пересборка) |
| `backup-htx` | Создание резервных копий данных PostgreSQL и Redis       |

## Проверка успешной инициализации

После успешной инициализации dev-контейнера вы должны:

1. Видеть сообщение "Инициализация dev-контейнера завершена" с зеленым статусом "УСПЕХ"
2. Иметь доступ к frontend через `http://localhost:3000`
3. Иметь доступ к backend через `http://localhost:8000`
4. Иметь базу данных `htx` в PostgreSQL

## Решение проблем

### Контейнеры не запускаются

Если контейнеры не запускаются, проверьте:

1. Запущен ли Docker демон/сервис
2. Есть ли конфликты портов (3000, 8000, 5432, 6379)
3. Логи контейнеров: `docker logs htxenterface_v2-postgres-1`

### Ошибки с базой данных

Если возникают ошибки с базой данных:

1. Проверьте доступность PostgreSQL: `docker exec -it htxenterface_v2-postgres-1 pg_isready`
2. Проверьте наличие базы данных htx: `docker exec -it htxenterface_v2-postgres-1 psql -U htx -d htxdb -c "\l"`
3. Создайте базу данных вручную: `docker exec -it htxenterface_v2-postgres-1 psql -U htx -d htxdb -c "CREATE DATABASE htx;"`

## Кастомизация
