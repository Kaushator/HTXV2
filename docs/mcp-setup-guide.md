# Настройка MCP Tools для проекта HTXEnterface_v2

## Что такое MCP (Model Context Protocol)

Model Context Protocol (MCP) - это протокол для взаимодействия с моделями искусственного интеллекта, который позволяет более эффективно передавать контекст и управлять взаимодействием с AI моделями. MCP Tools - это набор инструментов для реализации этого протокола в разработке.

## Установка MCP Tools

### 1. Установка MCP CLI

```bash
npm install -g @microsoft/mcp-cli
```

### 2. Проверка установки

```bash
mcp --version
```

## Настройка проекта

### 1. Структура директорий

MCP использует следующую структуру директорий:

```
.mcp/
  ├── config.json       # Основная конфигурация MCP
  ├── memory/           # Хранилище контекста и истории
  ├── plugins/          # Расширения для MCP
  └── templates/        # Шаблоны для генерации кода
```

### 2. Конфигурация MCP

Основной файл конфигурации `.mcp/config.json`:

```json
{
  "version": "1.0",
  "sources": [
    {
      "type": "code",
      "path": "./",
      "exclude": ["node_modules", ".git", "dist", "build"]
    },
    {
      "type": "documentation",
      "path": "./docs"
    }
  ],
  "ai": {
    "provider": "openai",
    "model": "gpt-4",
    "settings": {
      "temperature": 0.2
    }
  }
}
```

## Интеграция с DevContainers

### 1. Настройка общего контекста

Для интеграции MCP с несколькими контейнерами необходимо настроить общий доступ к данным MCP:

1. Добавьте в `docker-compose.extend.yml` и `docker-compose.fingpt.yml` монтирование общей директории:

```yaml
volumes:
  - ./mcp-shared:/shared/mcp
```

2. Создайте скрипт синхронизации в `.devcontainer/scripts/mcp-sync.sh`:

```bash
#!/bin/bash
mkdir -p /workspace/.mcp
mkdir -p /shared/mcp
rsync -avz --exclude='node_modules' --exclude='.git' /workspace/.mcp/ /shared/mcp/
rsync -avz /shared/mcp/ /workspace/.mcp/
```

3. Добавьте запуск скрипта синхронизации в postCreateCommand для обоих контейнеров.

### 2. Расширения VS Code для MCP

В файле devcontainer.json для обоих контейнеров должно быть добавлено расширение:

```json
"extensions": [
  "ms-mcp.mcp-vscode-extension"
]
```

## Использование MCP в проекте

### 1. Команды MCP CLI

- `mcp init` - инициализация проекта
- `mcp query <запрос>` - выполнение запроса к модели
- `mcp index` - обновление индекса контекста
- `mcp explain <файл>` - получение объяснения кода

### 2. Команды VS Code

В VS Code можно использовать команды:

- `/mcp query <запрос>` - выполнение запроса
- `/mcp explain` - объяснение выделенного кода
- `/mcp generate` - генерация кода по описанию

### 3. Интеграция с Specify

Specify интегрируется с MCP для улучшения контекста при разработке:

1. Используйте команду `/specify` в VS Code для создания новых спецификаций
2. MCP автоматически использует эти спецификации для контекста

## Советы по эффективной работе

1. Регулярно обновляйте индекс контекста командой `mcp index`
2. Используйте общие шаблоны для генерации кода
3. Документируйте код для лучшего понимания моделью
4. При работе с несколькими контейнерами запускайте скрипт синхронизации после значительных изменений

## Устранение проблем

1. При проблемах с синхронизацией контекста:

   - Проверьте права доступа к директориям
   - Запустите скрипт синхронизации вручную
   - Проверьте монтирование томов в docker-compose

2. При проблемах с расширением VS Code:
   - Переустановите расширение
   - Обновите VS Code до последней версии
   - Проверьте логи в Output -> MCP
