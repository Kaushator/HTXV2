# Инструкция по работе с Docker-контейнерами HTXV2

Эта документация описывает работу с Docker-контейнерами для проекта HTXV2. Здесь представлены скрипты для автоматизации управления контейнерами, их тестирования, обновления и создания резервных копий данных.

## Используемые контейнеры

Система HTXV2 использует следующие Docker-контейнеры:

- **postgres** - база данных PostgreSQL с расширением pgvector
- **redis** - хранилище ключ-значение Redis
- **backend** - API-сервер на FastAPI (Python)
- **frontend** - веб-интерфейс на Next.js (React)
- **fingpt** - сервис обработки данных с использованием FinGPT
- **dev** - контейнер для разработки

## Скрипты для управления контейнерами

В проекте представлены следующие скрипты для управления контейнерами:

### 1. test-containers.ps1

**Назначение**: проверка работоспособности Docker-контейнеров.

**Использование**:

```powershell
.\scripts\test-containers.ps1
```

**Функциональность**:

- Проверяет наличие и настройки Docker
- Валидирует конфигурацию в файле docker-compose.yml
- Проверяет статус всех контейнеров
- Тестирует доступность HTTP-сервисов (backend, frontend)
- Тестирует TCP-соединение с Redis
- Проверяет сетевые подключения контейнеров
- Анализирует использование ресурсов контейнерами
- Проверяет логи контейнеров на наличие ошибок

**Вывод**: результаты тестирования с указанием статуса (успех/ошибка) для каждой проверки.

### 2. update-containers.ps1

**Назначение**: обновление Docker-контейнеров.

**Использование**:

```powershell
.\scripts\update-containers.ps1
```

**Функциональность**:

- Останавливает контейнеры
- Удаляет старые Docker-образы
- Пересобирает образы с нуля (без использования кэша)
- Запускает контейнеры
- Проверяет статус запущенных контейнеров
- Проверяет и создает необходимые базы данных

**Вывод**: результаты обновления с указанием статуса для каждого этапа.

### 3. backup-data.ps1

**Назначение**: создание резервных копий данных.

**Использование**:

```powershell
.\scripts\backup-data.ps1
```

**Функциональность**:

- Создает директорию для резервных копий (если не существует)
- Проверяет, запущены ли необходимые контейнеры
- Создает резервную копию базы данных PostgreSQL
- Создает резервную копию данных Redis
- Создает резервную копию конфигурационных файлов

**Вывод**: информация о созданных резервных копиях, их расположении и размерах.

## Устранение распространенных проблем

### Проблема: Контейнер PostgreSQL не может найти базу данных "htx"

**Решение**:

```powershell
# Подключение к контейнеру PostgreSQL и создание базы данных
docker exec -it htxenterface_v2-postgres-1 psql -U htx -d htxdb -c "CREATE DATABASE htx;"
```

### Проблема: Контейнер не запускается или зависает

**Решение**:

```powershell
# Остановка и перезапуск конкретного контейнера
docker restart htxenterface_v2-[имя-контейнера]-1

# Или перезапуск всех контейнеров
docker-compose restart
```

### Проблема: Конфликт портов

**Решение**: изменить порт в docker-compose.yml и перезапустить контейнеры

```powershell
# Редактирование docker-compose.yml (изменить порты)
notepad docker-compose.yml

# Перезапуск контейнеров с новой конфигурацией
docker-compose down
docker-compose up -d
```

## Часто используемые команды Docker

```powershell
# Просмотр списка запущенных контейнеров
docker ps

# Просмотр логов контейнера
docker logs htxenterface_v2-[имя-контейнера]-1

# Запуск команды внутри контейнера
docker exec -it htxenterface_v2-[имя-контейнера]-1 [команда]

# Остановка всех контейнеров
docker-compose down

# Запуск всех контейнеров
docker-compose up -d
```

---

**Примечание**: Все скрипты разработаны для PowerShell (pwsh) и должны запускаться из корневой директории проекта HTXV2.
